# Инструкция по пересборке страницы каталога в вектор

Ты собираешь одну или несколько страниц A4-каталога «ЭнергоГрупп» заново: фото/градиент-фон
остаётся растровым, а ВЕСЬ ТЕКСТ пересобирается живым вектором шрифтом Onest поверх чистого
фона (старый впечатанный текст стирается, чтобы не было «двоения»).

## Геометрия
- Исходное изображение страницы: `page_images_150dpi/pgNN.jpg`, размер 1240×1754 px @150dpi.
- Итоговый PDF: A4 = 595.445669 × 841.691339 pt.
- baseline строки в спецификации = НИЖНЯЯ граница текстовой полосы (y1 из measure.py).
- width строки = измеренная ширина в px (x1 - x0 из measure.py). Кегль подбирается
  автоматически по ширине через метрики шрифта — вручную размер НЕ трогаем.

## Инструменты (уже в репозитории)
- `python3 scripts/measure.py page_images_150dpi/pgNN.jpg` — печатает TEXT BANDS (y0,y1,x0,x1)
  и ORANGE DOTS (cy,cx,r). ВНИМАНИЕ: на фото-фоне бывают ЛОЖНЫЕ полосы (детали фото). Отбирай
  только полосы, совпадающие с реальным текстом (сверяйся с картинкой глазами!).
- `scripts/rebuild.py` — модуль (шрифты, size_for, erase). Уже импортируется в build_deck.
- `scripts/build_deck.py` + `scripts/specs.py` — образец рабочей сборки (страницы 01,07,08,10
  уже собраны и проверены — бери их как эталон формата).

## Формат спецификации страницы (dict)
```python
dict(n=NN,
     erase=[(x0,y0,x1,y1), ...],       # зоны, где стереть старый СВЕТЛЫЙ текст
     contrast=38,                       # порог стирания (меньше = агрессивнее), по умолч. 38
     logo_erase=(x0,y0,x1,y1),          # прямоугольник старого логотипа (или не указывать)
     logo=(918,60,250),                 # (x0_px, top_px, w_px) для нового оранж. логотипа; None если логотипа нет
     title=[('ТЕКСТ', baseline, x, width), ...],   # строки заголовка (ExtraBold), одна кегль
     bullets=[(dot_cx, [('строка', baseline, x, width), ...]), ...],  # буллеты (Regular, оранж. точка)
     body=[('текст', baseline, x, size_pt, 'Onest-Regular', (r,g,b)), ...],  # свободный текст
     dot_r=11)                          # радиус оранжевой точки (px), по умолч. 11
```
Цвета: белый текст = (240/255,240/255,246/255); оранжевый = (252/255,144/255,43/255).

## Логотип
- Ассет: `assets/logo_orange.png` (оранжевый, прозрачный фон). Ставится в правый верх.
- Стандартное место: `logo=(918, 60, 250)`. Старый логотип на фоне стирай через `logo_erase`
  прямоугольником (обычно (915,55,1170,235)).
- Если на странице логотипа НЕТ (по описанию) — `logo=None`, `logo_erase` не указывай.

## Как проверять (ОБЯЗАТЕЛЬНО)
1. Впиши свою спецификацию во временный скрипт (НЕ трогай общий specs.py — избегаем гонок).
   Скопируй функцию build() из build_deck.py или импортируй её.
2. Собери PDF, отрендери в PNG:
   `python3 -c "import fitz; fitz.open('build/pageNN.pdf')[0].get_pixmap(dpi=120).save('/tmp/NN.png')"`
3. ОТКРОЙ PNG инструментом Read и СРАВНИ с оригиналом `page_images_150dpi/pgNN.jpg`:
   - текст совпадает по словам, переносам, положению;
   - нет «двоения» (старый текст не проступает) — если проступает, расширь erase-зону / понизь contrast;
   - заголовок и буллеты на своих местах, кегль похож на оригинал;
   - логотип чистый.
4. Итерируй, пока не будет чисто.

## Что ВЕРНУТЬ
Верни ТОЛЬКО готовые python-dict спецификации (по одному на страницу) в блоке ```python ...```,
дословно в формате выше, с уже проверенными координатами. Никаких пояснений внутри блока.
Не полагайся на запись в общие файлы — важен именно текст спецификаций в ответе.
