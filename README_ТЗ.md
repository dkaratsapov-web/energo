# ТЗ — допечатная пересборка каталога «ЭнергоГрупп. Комплексное строительство 2025»

## 0. Контекст и цель
Исходный каталог (32 стр., A4) пришёл **полностью сведённым в растр 150 dpi**: каждая
страница — одна плоская картинка, где фото, текст и логотипы «впечатаны». Типография
отклонила файл: **текст и фото пикселят** при печати. Векторного исходника (Figma/AI/CDR/
INDD/PPTX) НЕТ.

**Задача (вариант «Б»):** пересобрать каждую страницу заново —
- **весь текст → живой вектор** шрифтом **Onest** (заголовки ExtraBold, тело Regular);
- **плашки/градиенты → вектор**;
- **фото → в высоком разрешении** (HD-оригиналы от клиента; «нормальные» оставляем как есть,
  «плохие» — клиент присылает HD по запросу);
- **логотип → вектор** (ждём файл логотипа от клиента);
- финал — **PDF/X-1a, CMYK (FOGRA39), вылеты ≥4 мм, БЕЗ меток обреза, 300 dpi для растра**.

Шрифт Onest подтверждён: при пересборке ширина строк совпадает с оригиналом до 1–2 px
(значит, кегль/трекинг/начертание верные).

---

## 1. Геометрия и калибровка (ВАЖНО, использовать везде)
- Лист A4 = **595.445669 × 841.691339 pt**.
- Постраничный фон = **1240 × 1754 px @150 dpi**.
- Перевод px → pt:
  - `sx = 595.445669 / 1240 = 0.480199`
  - `sy = 841.691339 / 1754 = 0.479870`
  - `X(px) = px * sx`
  - `Y(px) = PH - py * sy`  (в PDF начало координат снизу-слева!)
- **Кегль подбирается ТОЧНО по метрикам** (без ручных подгонок):
  `size = (target_width_px * sx) / pdfmetrics.stringWidth(text, font, 1.0)`
- Шрифты: `fonts/Onest-{Regular,Medium,SemiBold,Bold,ExtraBold}.ttf`
  (сделаны из вариативного `Onest-VF.ttf`, оси wght 400/500/600/700/800).

## 2. Цвета (уточнять пипеткой по каждой странице)
- Заголовок (белый): ~ `(240,240,246)`
- Маркеры-буллеты (оранжевый бренд): `(252,144,43)`
- Фиолетовая плашка (вертикальный градиент): верх `(67,70,103)` → низ `(87,88,144)`
- ⚠️ На некоторых страницах текст ТЁМНЫЙ на светлом фоне (напр. стр. 22 — дорога).

## 3. Метод пересборки одной страницы (без «двоения»!)
Двоение = старый впечатанный текст проступает из-под нового вектора. Поэтому порядок такой:
1. **Стереть старый текст с фона**: `build_page.erase_old_text(...)` — inpaint по маске
   белого текста в заданных прямоугольниках. (Для тёмного текста — инвертировать порог.)
   Для текста на СПЛОШНОЙ плашке проще перекрасить плашку/градиент заново.
2. **Наложить вектор**: `build_page.build_page(...)` — заголовок + буллеты (см. пример
   внизу `build_page.py`, это реальная стр. 7).
3. **Фото**: «нормальные» оставляем; «плохие» — подставляем HD от клиента (crop под слот).

Скрипты: `scripts/measure.py` (снять координаты/ширину), `scripts/build_page.py` (собрать),
`scripts/make_print_pdfx.py` (финальный печатный конвейер).

## 4. Финальный печатный конвейер (текст НЕ растрировать!)
`scripts/make_print_pdfx.py`:
1. `combine()` — склеить 32 векторные страницы по порядку.
2. Вылеты 4 мм: **лучший способ** — собирать страницы сразу размером trim+bleed
   (фон-картинку расширять reflect-паддингом на 4 мм), TrimBox = внутренний прямоугольник.
3. `to_pdfx()` — `gs -dPDFX` + `color/PDFX_def.ps` → CMYK PDF/X-1a, шрифты встраиваются,
   `-dPassThroughJPEGImages=true` сохраняет CMYK-JPEG фото.
4. `add_xmp()` — XMP-метаданные PDF/X (pikepdf).
- `color/PDFX_def.ps` ссылается на `color/FOGRA39.icc` — **путь к .icc сделать абсолютным!**
- Проверка результата: `pdfimages -list` (должно быть cmyk, 0 rgb), TrimBox=210×297 мм,
  `GTS_PDFXConformance=PDF/X-1a:2001`, OutputIntent=FOGRA39.

## 5. Требования типографии (их документ)
- Размер страницы = размеру продукции (A4), постранично. [OK]
- Цвет **CMYK** (цифровая печать 4 краски). [OK] FOGRA39.
- Растр **300 dpi**, спецэффекты — в растр. ⚠️ Следить за dpi подставляемых фото!
- Вылеты «на вылет» ≥ 4 мм. [OK]
- Шрифты «в кривые» → у нас вектор Onest со встроенными шрифтами (PDF/X встраивает).
  Если типография требует именно аутлайны — можно перевести текст в кривые
  (`gs -dNoOutputFonts` НЕ годится; используйте перевод в кривые в редакторе или
  mutool/qpdf-иное; проще — оставить встроенные шрифты, PDF/X это допускает).
- Логотип/знак — вектором (ждём файл логотипа от клиента).
- Файл БЕЗ меток обреза (спуск полос делают сами в CorelDraw). [OK]

## 9. Порядок работы для продолжения
1. Для каждой [TODO]-страницы: `measure.py` → снять полосы текста и маркеры → занести в `build_page`.
2. Стереть старый текст (`erase_old_text`), собрать вектор (`build_page`).
3. Сложную графику (график стр.4, иконки стр.5, карта 28–29, лого партнёров стр.30)
   — перерисовать вектором или подставить векторные ассеты от клиента.
4. Подставить HD-фото туда, где решено заменять.
5. Собрать всё (`make_print_pdfx.py`): вылеты 4 мм → gs PDF/X-1a (CMYK FOGRA39) → XMP.
6. Проверить `pdfimages -list`, боксы, конформанс.
